### 사용 방법

**1. 플레이리스트 추가**
가운데 **+ 버튼**을 눌러 새로운 플레이리스트를 생성합니다.

**2. 곡 추가 방법**
원하는 유튜브 영상을 다양한 방식으로 추가할 수 있습니다.

* 검색을 통해 곡 추가
* URL 직접 추가 (유튜브 영상 링크 복사 후 등록)
* 유튜브 썸네일 또는 URL 드래그로 추가

**3. 기능 버튼 설명 (왼쪽부터 순서대로)**

* **셔플 버튼**
  현재 플레이리스트의 곡 순서를 무작위로 섞습니다.
  (DB에는 저장되지 않으며, 셔플 상태에서 곡을 추가하면 새 곡은 맨 뒤에 추가됩니다)

* **전체 플레이리스트 재생 버튼**
  마지막 곡 재생 후
  → 다음 플레이리스트로 넘어갈지
  → 현재 플레이리스트를 반복할지 설정할 수 있습니다.

* **수면 타이머 버튼**
  설정한 시간 후 자동으로 재생이 정지됩니다.

* **플레이리스트 공유 / 적용 버튼**
  플레이리스트를 파일로 저장하여 내보낼 수 있습니다.
  해당 파일을 통해 다른 계정에서 적용하거나 백업 용도로 활용할 수 있습니다.

**4. 현재 곡 위치로 이동 기능**
노래 재생 시 오른쪽 위에 표시되는 작은 썸네일을 클릭하면 해당 곡의 위치로 이동할 수 있습니다.

**5. 모달창 닫기**

* `ESC` 키 입력
* 모달창 바깥 영역 클릭
  → 두 방법 모두 모달을 닫을 수 있습니다.


# Music Player

YouTube API를 활용해 개인화된 음악 플레이리스트를 관리하는 웹 애플리케이션입니다.  
유저별로 독립적인 Playlist 환경을 제공하며, 실시간으로 곡을 추가하고 순서를 관리할 수 있습니다.
각 유저는 Oauth, Session 방식으로 로그인을 관리합니다.

---

## Problem

음악 플레이어 웹 애플리케이션을 구현하면서 다음과 같은 문제를 겪었습니다.

- 유튜브 iframe을 단순 삽입할 경우 플레이어 상태 제어가 어려움
  - 영상 종료 감지 불가
  - 자동 다음곡 재생 불가
  - 백그라운드 상태 제어 불안정

- Next.js Middleware와 Prisma 간 Runtime 충돌
  - Middleware는 Edge Runtime 기반
  - Prisma는 Node.js 환경 필요 → 직접 사용 불가

- 유저별 Playlist 데이터를 안정적으로 관리해야 하는 요구사항
  - 사용자별 데이터 분리 및 관계형 구조 필요

- 첫 협업작업이라 커밋/브랜치를 나누는 것과 pr 설명에 대한 어려움
---

## Approach

문제를 해결하기 위해 다음과 같은 방향으로 접근했습니다.

- 단순 iframe 방식에서 벗어나 YouTube Iframe API 기반 구조로 전환
- Runtime 충돌 문제를 해결하기 위해 인증 로직과 DB 로직 분리
- Prisma를 활용하여 관계형 데이터 구조 설계
- 클라이언트 상태와 서버 데이터를 명확히 분리하여 관리
- 현 카카오 개발자 친구의 코드리뷰 및 피드백

---

## Action

### 1. YouTube 플레이어 제어 구조 개선
- iframe 직접 삽입 방식 → YouTube Iframe API 기반으로 변경
- `window.YT.Player` 객체를 활용하여 플레이어 인스턴스 생성
- 이벤트 리스너를 통해 곡 종료 시점 감지 및 자동 다음곡 재생 구현

### 2. Runtime 충돌 해결 (Middleware vs Prisma)
- 인증 설정을 `auth.config.ts`(Edge 환경)와 `auth.ts`(DB 접근)로 분리
- Middleware에서는 DB 접근 없이 가벼운 인증 체크만 수행
- 서버 환경에서만 Prisma Client를 사용하도록 구조 개선

### 3. 데이터 구조 설계
- Prisma + PostgreSQL 기반 관계형 데이터 모델링
- User - Playlist - Song 구조로 설계하여 유저별 데이터 독립성 확보

### 4. 핵심 기능 구현
- 카카오/구글 OAuth 기반 로그인 (Auth.js)
- Playlist 및 곡 추가 / 삭제
- Drag & Drop 기반 곡 순서 변경
- 실시간 상태 반영
- 자신의 플레이리스트를 공유 또는 백업 가능

---

## Performance

- 플레이어 상태를 직접 제어할 수 있게 되어 사용자 경험 개선
- Runtime 분리를 통해 Middleware와 DB 간 충돌 문제 해결
- 유저별 데이터 구조를 명확히 하여 확장성과 유지보수성 향상
